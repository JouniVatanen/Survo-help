Earlier written observations may be protected against changes by
mistake by adding the key word PROTECTED after file name in the
FILE statement, e.g.
    FILE FILE1 PROTECTED;
A given number of observations may be protected by stating the
number of protected observations after the keyword:
    FILE FILE1 PROTECTED 266;
At this moment the state of protection cannot be changed during
the MFEDIT session.

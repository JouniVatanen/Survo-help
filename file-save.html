File save

Description

FILE SAVE saves contents of a text file to Survo data file.

Usage

FILE SAVE 'text file' TO NEW 'Survo data file'

'text file' is the whole name of the text file that you want to save. Use without '-characters and with the correct file extension 'Survo data file' is name of the file where the data will be saved. Use without '-characters. NEW is optional and deletes the existing Survo data file.

Parameters

NAMES
NAMES=names_line gives he line for the variable names in Survo data file. Default (if no NAMES given): variable names X1, X2, ... are used. NAMES=DEFAULT maps the columns of a text file to the active variables of Survo data file.
DELIMITER
DELIMITER=char(number) separates the columns of the text file. Default: one or more blank (space) characters. Possible values are denoted by char(i), i=1,...,255. Keyword TAB may be used instead of char(9).
FIRST
FIRST=first_line. The first line to be copied from the text file. Default: 1.
LAST
LAST=last_line. The last line to be copied from the text file. Default: last line of the text file.
MISSING
MISSING=string treats string as a missing value. Default: spaces and/or '-'.
SELECT='Boolean expression'
selects observations satisfying the conditions given by Boolean expression=A+B+C+... (read A or B or C or ...) or A*B*C*... (read A and B and C and ...). Each condition A,B,C,... must be given as a specification of the form
A1='variable','lower limit','upper limit' or
A1='string variable':'case1','case2',...
Any words can be used in place of A,B,C.
Example: SELECT=North*Over10000+South*Over20000 North=Province:OUL,LAP South=Province:UUS,KYM,TUR Over10000=Popul,10000,500000 Over20000=Popul,20000,500000
selects cases where either Province is OUL or LAP and Popul =10000 or Province is UUS or KYM or TUR and Popul =20000
The SELECT specification can also be written with negations (a `!' in front of a Boolean expression) and with parentheses. For details read more about select observations and/ or variables.
VARS='list of variables'
is a specification which selects the variables into statistical analysis. Another way to do that is to use a MASK specification, but VARS always overrides MASK and permanent activation status in the data file.
For example, VARS=Height,Weight,Test1 selects the variables Height,Weight and Test1 which all are activated by 'A' in the current data file; other variables will be passive. VARS=ALL means that all variables are to be selected. VARS=ALL,-Name,-Test1 means that all variables except Name and Test1 are to be selected. The task of each variable can be given in parentheses in the VARS list.
For example, in the LINREG operation VARS=Height(X),Weight(X),Test1(Y),Residual(R) selects the variables and indicates their roles in the analysis as well. The influence of VARS is only temporary and does not change the activation status in the data file.
MASK
MASK without parameters is an operation which leads to the activation procedure for the fields of any Survo data file. After return from MASK procedure to normal edit mode the current line will contain a MASK specification of the form MASK=--XX-A-YX-R---AA- (for example) indicating the activated fields in the current data file. If a MASK specification appears in the current subfield, when a statistical operation like CORR,LINREG,TAB etc. is activated, it determines the roles of the variables in the analysis. However, a possible VARS specification always overrides MASK in statistical operations. By activation of a MASK line with a given mask string (MASK=--XX-A-- , for example) the selection of the fields determined by that string will be shown for a selected file and this selection may be adjusted. After return to edit mode again the MASK line will show the latest selection. Since a data file can contain several optional mask columns (default number is 7), any of them can be used to indicate the selected vari- ables by giving its number #1,#2,#3,,, as MASK=#7. A partial selection according to a mask column is also possible. For example, MASK=#7(XY) selects the fields indicated by X or Y in mask column 7. Activation of a MASK line of this type will lead to a file activation display of that mask column for additional changes.
MAXFIELDS
MAXFIELDS=# of fields defines maximum number of fields in the text file. Default: 300.
ENCODING
encoding of the text file spesifies, how to encode the text file. This spesification is used only if the system doesn't regognize encoding. ENCODING=UTF-8 encodes UTF-8 text files. Default: DEFAULT.
> R
> R'R data frame' transforms R data frame to Survo data. Use without '-characters.
SKIP
SKIP=list_of_characters skips unwanted charcters.
FIELDS
FIELDS specification makes difficult text file structures readable to a Survo data files. For more information check FILE SAVE with FIELDS.
MATCH
Only to use with FIELDS specification. For more information check FILE SAVE WITH FIELDS.
SKIP_ERRORS
Does not interrupt saving the text file when the line has too few fields. SKIP_ERRORS=1 deletes incomplete lines. SKIP_ERRORS=2 creates a data record from an incomplete line and assumes that missing fields are at the end of the text line
CHECK_VAR_NAMES
CHECK_VAR_NAMES=0 treats variable names as is instead of altering the 8th character if duplicate variable names are encountered.
REMOVE_QUOTES
REMOVE_QUOTES=1 converts double quotes within text to ''.
FORMAT
FORMAT=WORD,'variable','delimiters' saves the words of a text file to a selected variable. Default for delimiters: ' ,.:;?!-'.
FORMAT=CHAR,'variable','length' save the character of a text file to a selected variable. Default for length: 1. Spaces are skipped. FORMAT=0 avoids collison with wrong format-lists like NOFORMAT=1.
NOFORMAT
NOFORMAT=1 avoids collison with wrong format-lists like FORMAT=0.
FILTER
FILTER=conversion file converses a text file to a Survo data file with specific conversion rules. For making code conversion files check code conversion.FILTER=#DOTCOMMA FILTER=#COMMADOT replaces dots with commas and vice versa. FILTER=#SEMICOMMA replaces semicommas with commas.
NOFIELDS
NOFIELDS avoids collision with wrong field-lists.
NEWSPACE
NEWSPACE=m,n creates a Survo data file with space for m records and n variables. NEWSPACE=0,0 creates a Survo data file with no extra space.
DEC
DEC=character specifies the decimal character used. Default: . (dot).
MODEL
MODEL=# of a line allows to give 'model-line'.???

Details

Before creating the Survo data file, FILE SAVE scans the entire text file. It detects whether the field is a string or a numeric field. If it appears as the first line in the text file and the labels are strings (not starting by digits), these labels will appear as names of fields in the new Survo data file.

Names of variables in Survo data files consist of max. 8 character identifier and of its extension (more accurate description, format, etc.). The identifier part must be started by a letter and other characters are typically letters and numbers. Characters like +-*/()[]{},!"'? should be avoided.

The maximum line length in the text file is at least 100 000. Longer records has to be divided into several lines.

Examples

FILE SAVE data1.txt TO data / DELIMITER=TAB
..........


FILE SAVE data1.txt TO data / NAMES=1 FIRST=2 LAST=10
..........


FILE SAVE data1.txt TO data / SKIP=";|
..........


FILE SAVE data1.txt TO data / DELIMITER=TAB NAMES=1 FIRST=2
